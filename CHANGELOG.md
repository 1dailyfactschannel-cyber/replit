# Changelog — последние изменения

## Спринты и Agile

- **Спринты в проектах** — возможность создавать спринты внутри проектов, назначать даты начала/окончания, статус (запланирован, активен, завершён).
- **Бэклог** — отдельный раздел для задач без спринта. Переключение между бэклогом и спринтами в боковой панели проекта.
- **Привязка задач к спринту** — в карточке задачи можно выбрать спринт или оставить задачу в бэклоге.
- **Избранные спринты** — возможность добавлять спринты в избранное звёздочкой. Избранные спринты отображаются в главном сайдбаре рядом с избранными проектами.
- **Прямой переход к спринту** — при клике на избранный спринт в сайдбаре происходит навигация на проект с автоматическим выбором спринта через URL-параметр `?sprintId=`.

## Отчёты и аналитика

- **Отчёт «Нагрузка» (Team Workload)** — новая вкладка в разделе «Отчёты»:
  - Сводные KPI-карточки: общее количество задач, просроченные задачи, средняя загрузка, сотрудники в зоне риска.
  - Список сотрудников с прогресс-барами загрузки по статусам задач (в работе, на проверке, просрочено).
  - Столбчатая диаграмма (Recharts) с распределением задач по статусам для каждого сотрудника.
  - Уровни риска: низкий, средний, высокий (на основе количества активных задач).

## Зависимости задач

- **Связи между задачами** — возможность создавать зависимости задач (блокирует / зависит от).
- Визуальное отображение связей в деталях задачи.

## Роли и права доступа (RBAC)

- **Расширенная система ролей** — гибкая настройка прав доступа через роли.
- **Системные роли** — защита системных ролей (Владелец, Администратор, Менеджер, Сотрудник, Гость) от удаления.
- **Удаление роли с переносом пользователей** — при удалении кастомной роли можно переназначить всех пользователей на другую роль.
- **Сброс пароля сотрудника** — администратор может сбросить пароль любому сотруднику со страницы команды.
- **Назначение владельца проекта** — возможность назначить владельца при создании/редактировании проекта.
- **Проверка прав доступа** — централизованная система проверки прав через `usePermission` и middleware на бэкенде.

## Уведомления и коммуникации

- **Система уведомлений** — real-time уведомления через WebSocket + Socket.IO.
- **Упоминания в комментариях** — `@username` в комментариях к задачам отправляет уведомление.
- **Ответы на сообщения** — возможность ответить на конкретное сообщение в чате.
- **Комнаты команды (Team Rooms)** — групповые чаты для команд.
- **Звуковые уведомления** — аудио-оповещения о новых уведомлениях.
- **Telegram-уведомления** — интеграция с Telegram для упоминаний и системных событий.
- **Email-уведомления** — сервис отправки email (календарные напоминания и др.).

## Звонки и WebRTC

- **Аудио/видео звонки** — интеграция Mediasoup для peer-to-peer звонков.
- **Экран звонка** — оверлей с управлением микрофоном, камерой и завершением звонка.

## Магазин и начисления

- **Магазин (Shop)** — каталог товаров с возможностью запроса на покупку.
- **Система начислений** — начисление баллов/валюты за выполнение задач.
- **Правила начислений (Accrual Rules)** — настройка условий автоматических начислений.

## Проекты и доски

- **Рабочие пространства (Workspaces)** — группировка проектов по рабочим пространствам.
- **Избранные проекты** — добавление проектов в избранное с отображением в сайдбаре.
- **Описание колонок** — возможность добавлять описание к колонкам Kanban-доски.
- **Типы задач** — категоризация задач по типам.
- **Архивирование задач** — возможность архивировать задачи вместо удаления.
- **Кастомные статусы** — создание собственных статусов задач.

## Интеграции

- **Яндекс Календарь** — интеграция с Yandex Calendar для синхронизации событий.
- **Telegram-бот** — автоматическая настройка вебхука при старте сервера.

## Безопасность и администрирование

- **Аудит-логи** — таблица для хранения истории действий пользователей.
- **Гостевые приглашения** — система приглашений гостей в проект/пространство.
- **Приглашения по email** — отправка приглашений с указанием пригласившего.
- **Подтверждение смены пароля** — email-подтверждение при изменении пароля в настройках безопасности.
- **CSRF-защита** — middleware CSRF для защиты API.
- **Поддержка bcrypt** — исправлена несовместимость scrypt/bcrypt при авторизации.

## Настройки пользователя

- **Рабочее время** — настройка времени начала и окончания рабочего дня.
- **Пользовательские настройки** — key-value хранилище настроек (избранное, активный проект, свёрнутые проекты и др.).

## Инфраструктура и деплой

- **Jenkins CI/CD** — автоматическая сборка и деплой при коммите в `main`.
- **Docker-контейнеры** — сервер и клиент собираются отдельно.
- **Nginx-проксирование** — проксирование API, auth и WebSocket через nginx.
- **Поддержка Socket.IO через nginx** — добавлен location `/socket.io/` для проксирования WebSocket.
- **Исправление Dockerfile.client** — добавлены `python3` и `build-essential` для сборки нативных npm-пакетов (mediasoup).

## Багфиксы и улучшения

- **Исправление кеша React Query** — корректная инвалидация кеша после обновления спринта задачи.
- **Защита от `null` в `userSettings`** — оборонительное программирование при работе с массивами настроек.
- **Кастомный `queryFn`** для запросов с query-параметрами (обход `queryKey.join("/")`).
- **Все UI-тексты на русском** — приведение интерфейса к единому языку.
