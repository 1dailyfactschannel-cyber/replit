# Интеграция Яндекс Календарь

## Обзор

Интеграция позволяет пользователям синхронизировать события из их личного Яндекс Календаря с календарем TeamSync. События отображаются в интерфейсе с цветами из Яндекса.

## Функциональность

✅ **Персональная OAuth-авторизация** - каждый пользователь подключает свой календарь  
✅ **Односторонняя синхронизация** - только импорт из Яндекса (Яндекс → TeamSync)  
✅ **Повторяющиеся события** - поддержка RRULE (разворачиваются в календаре)  
✅ **Push-уведомления** - о предстоящих встречах и изменениях событий  
✅ **Звуковые уведомления** - при обновлениях и напоминаниях  
✅ **Email-уведомления** - HTML-письма о предстоящих встречах  
✅ **Импорт цветов** - сохраняются цвета событий из Яндекса  
✅ **Автосинхронизация** - каждые 10 минут  

## Архитектура

```
Пользователь → OAuth → Яндекс → CalDAV API → Бэкенд (синхронизация)
                                                    ↓
                                            PostgreSQL
                                                    ↓
WebSocket ← Нотификации ← Проверка уведомлений (каждую минуту)
      ↓
Frontend (Календарь)
```

## Настройка

### 1. Создание OAuth приложения в Яндексе

**Шаг 1:** Перейдите на https://oauth.yandex.ru/

**Шаг 2:** Нажмите **"Создать приложение"**

**Шаг 3:** Выберите тип приложения:
- ✅ **Веб-сервисы** (Web services)
- ❌ Не выбирайте "Для авторизации" или "Для доступа к API"

**Шаг 4:** Заполните форму:
- **Название**: TeamSync Calendar
- **Описание**: Синхронизация календаря с TeamSync
- **Иконка** (опционально)

**Шаг 5:** Callback URL для перенаправления:
```
http://localhost:3000/api/integrations/yandex-calendar/callback
```
Для продакшена замените `localhost:3000` на ваш домен.

**Шаг 6:** Настройте доступы (Scopes):
Перейдите в раздел **"Доступы"** и отметьте:
- ✅ **Доступ к Яндекс.Календарю** (Calendar API)
  - ✅ Чтение календарей
  - ✅ Чтение событий
- ✅ **Доступ к Яндекс.Паспорту**
  - ✅ Доступ к email адресу

**Шаг 7:** Сохраните приложение

**Шаг 8:** Скопируйте учетные данные:
- **Client ID** - короткая строка (например: `a1b2c3d4e5f6789`)
- **Client Secret** - длинная строка
  - ⚠️ **Важно**: Client Secret показывается только один раз! Сохраните его сразу.
  - Если потеряли - нужно создать новое приложение

### 2. Настройка переменных окружения

Добавьте в `.env`:

```env
YANDEX_CLIENT_ID=your_client_id_here
YANDEX_CLIENT_SECRET=your_client_secret_here
APP_URL=http://localhost:3000  # или ваш домен
```

### 3. Добавление звуковых файлов

Скачайте или создайте звуковые файлы и поместите в `client/public/sounds/`:

- `reminder.mp3` - звук напоминания
- `calendar_update.mp3` - звук обновления

## API Endpoints

### OAuth и управление

```
GET  /api/integrations/yandex-calendar/auth      - Получить OAuth URL
GET  /api/integrations/yandex-calendar/callback  - Callback от Яндекса
GET  /api/integrations/yandex-calendar/status    - Статус подключения
DELETE /api/integrations/yandex-calendar         - Отключить интеграцию
POST /api/integrations/yandex-calendar/sync      - Ручная синхронизация
```

### Получение событий

```
GET /api/calendar/yandex-events?from=ISO&to=ISO - Получить события
```

## WebSocket События

Сообщения от сервера к клиенту:

```typescript
// Синхронизация завершена
socket.on('calendar:yandex:sync', (data) => {
  // { added: number, updated: number, deleted: number }
})

// Изменение события
socket.on('calendar:yandex:event', (data) => {
  // { type: 'new_event' | 'event_changed' | 'event_deleted', event: {...} }
})

// Напоминание
socket.on('calendar:reminder', (data) => {
  // { title, body, icon, tag, requireInteraction }
})

// Звук
socket.on('notification:sound', ({ type }) => {
  // type: 'reminder' | 'calendar_update'
})
```

## Как это работает

### Подключение

1. Пользователь нажимает "Подключить" на странице Управление → Интеграции
2. Открывается popup с OAuth Яндекса
3. Пользователь авторизуется и дает разрешение
4. Сервер получает токены и сохраняет их
5. Происходит начальная синхронизация

### Синхронизация

1. Cron job запускается каждые 10 минут
2. Для каждого активного подключения:
   - Проверяется срок токена (обновляется если нужно)
   - Запрашиваются события из Яндекса за период (-3 месяца, +6 месяцев)
   - Сравниваются с локальными событиями
   - Создаются/обновляются/удаляются события в БД
   - Отправляются WebSocket уведомления об изменениях

### Отображение

1. При открытии календаря загружаются:
   - Локальные события
   - События из Яндекса
2. Повторяющиеся события разворачиваются с помощью RRULE
3. События отображаются с цветной полоской слева
4. При клике открывается детальный просмотр (read-only)

### Уведомления

1. Каждую минуту проверяются напоминания
2. Если время напоминания наступило:
   - Отправляется Push notification (если разрешено)
   - Воспроизводится звук
   - Отправляется email
3. Отправленные уведомления записываются в БД (чтобы не дублировать)

## Отладка

### Логи

```bash
# Проверка статуса интеграции
GET /api/integrations/yandex-calendar/status

# Ручная синхронизация
POST /api/integrations/yandex-calendar/sync

# Проверка событий
GET /api/calendar/yandex-events?from=2024-01-01&to=2024-12-31
```

### Проблемы

1. **Ошибка OAuth**: Проверьте CLIENT_ID и CLIENT_SECRET
2. **Не приходят уведомления**: Проверьте разрешение на уведомления в браузере
3. **Не воспроизводится звук**: Добавьте звуковые файлы в `client/public/sounds/`
4. **Ошибка синхронизации**: Проверьте логи сервера

## Безопасность

- Токены хранятся в PostgreSQL
- OAuth использует state parameter для защиты от CSRF
- Доступ только к чтению календарей (scope: calendar:read)
- Автоматическое обновление access_token через refresh_token
